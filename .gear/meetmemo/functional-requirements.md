# 機能要件一覧

| id | functionName | description | relatedRequirement | priority |
| --- | --- | --- | --- | --- |
| FR-001 | ログイン・ログアウト機能 | システム利用者がID/パスワードを用いてログイン・ログアウトを行う画面およびAPI。従業員、マネージャー、HR管理者のロールに応じたメニュー制御（RBAC）を含む。※要確認：既存のActive DirectoryやSSO（シングルサインオン）との連携要件の有無は未定義。 | BR-011, BR-014, BR-016 | 高 |
| FR-002 | 有給休暇残日数表示機能 | 従業員がログイン後、自身の現在の有給休暇残日数を画面上で確認できる機能。申請画面およびダッシュボードに表示する。 | BR-004, BR-006 | 高 |
| FR-003 | 休暇申請登録・送信機能 | 従業員が休暇タイプ（有給、病欠等）、開始日、終了日、理由を入力し、必要に応じてファイルを添付して申請を登録する画面およびAPI。過去日付の申請不可チェック、終了日≧開始日チェック、有給残日数不足チェックのビジネスルール検証をリアルタイムで行う。※要確認：添付ファイルの容量制限やファイル形式、休暇タイプマスタの構成は未定義。 | BR-001, BR-002, BR-003, BR-004, BR-005, BR-007 | 高 |
| FR-004 | 自己申請履歴・ステータス一覧照会機能 | 従業員が自身が過去に申請したすべての休暇履歴と、現在の承認ステータス（申請中、承認済、却下済など）を一覧形式で検索・確認できる画面。 | BR-006 | 高 |
| FR-005 | 申請キャンセル・変更申請機能 | 従業員が自身が申請したデータについて、承認前は即時キャンセル、承認後は特定のルールの下でキャンセル・変更を行える機能。※要確認：承認済みの休暇申請のキャンセル・変更をシステム上許可するか、およびその申請可能期限等のビジネスルールは未定義。 | BR-008 | 中 |
| FR-006 | 承認対象申請一覧・詳細表示機能 | マネージャーがログインした際、自身が直属の上長として紐付けられているメンバーから提出された承認待ちの休暇申請を一覧および詳細画面で確認できる機能。※要確認：直属のマネージャーとメンバーの紐付けロジックおよび組織マスタのデータソース連携方法は未定義。 | BR-009, BR-011 | 高 |
| FR-007 | 休暇申請承認・却下実行機能 | マネージャーが申請の詳細を確認し、承認または却下のアクションを実行する機能。却下選択時は、却下理由のテキスト入力を必須とするバリデーション処理を含む。※要確認：マネージャー長期不在時の代理承認・代替承認ルートの制御ルールは未定義。 | BR-010, BR-012, BR-013 | 高 |
| FR-008 | 全社休暇申請実績検索・照会機能 | HR管理者が、システム内の全従業員の休暇申請データ、承認状況、および有給休暇残日数を横断的に検索・参照できる画面。部門別、従業員別、取得期間等の条件でフィルタリングが可能。HR管理者による代理承認は行えないよう画面制御する。 | BR-014, BR-016 | 高 |
| FR-009 | 月次休暇実績レポート抽出・出力機能 | HR管理者が指定した月の休暇取得実績データ（全社・部門単位）を一括抽出してファイル出力する機能。※要確認：出力フォーマット（CSV、Excel、PDF等）の詳細、および月次データ確定のための「締め処理」バッチ/運用の有無は未定義。 | BR-015 | 高 |
| FR-010 | 管理者向け休暇データ直接調整機能 | HR管理者が、従業員の有給休暇残日数の手動調整や、承認完了後の急な休暇スケジュール変更をシステム上で直接修正・メンテナンスできる機能。※要確認：この直接修正権限をHR管理者に付与するか、または従業員に再申請を強制するかの実運用フローは未定義。 | BR-017 | 中 |
| FR-011 | 有給休暇自動付与バッチ | 従業員の入社日や勤続年数に応じて、有給休暇日数を自動で計算・付与し残日数を更新するバッチ処理。※要確認：有給付与の計算ロジック（基準日、付与日数テーブル等）は未定義。 | BR-004 | 中 |
| FR-012 | 通知メール自動送信機能 | 申請登録時（宛先：直属マネージャー）、および承認/却下確定時（宛先：申請者本人、却下時は理由を記載）に自動で通知メールを配信する非同期バックエンド処理。※要確認：外部SMTPサーバー、または連携するメール送信用SaaSの仕様は未定義。 | BR-001, BR-009, BR-010, BR-013 | 高 |
| FR-013 | 操作・監査ログ記録機能 | ログイン・ログアウト、休暇申請の登録、承認・却下等のステータス遷移を、実行ユーザーID、実行日時、操作内容とともにデータベースへ記録するバックエンド処理。※要確認：監査ログ用の別データベース/スキーマ分割要否、および具体的なログ保存期間は未定義。 | BR-012 | 高 |
