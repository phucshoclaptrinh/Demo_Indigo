# 業務要件一覧

| id | category | requirement | priority | description |
| --- | --- | --- | --- | --- |
| BR-001 | 休暇申請管理 | 休暇申請の登録 | 高 | 従業員が、休暇タイプ、開始日、終了日、理由を入力して休暇申請を登録し、ワークフローを開始できること。 |
| BR-002 | 休暇申請管理 | 過去日付での申請制限 | 高 | システム上の業務ルールとして、従業員は本日より過去の日付を休暇開始日または終了日とする申請を行えないこと。 |
| BR-003 | 休暇申請管理 | 申請期間の妥当性チェック | 高 | 休暇申請時、終了日が開始日と同じ、または開始日より後の日付（終了日 ≧ 開始日）であることをシステムが強制し、誤入力を防ぐこと。 |
| BR-004 | 休暇申請管理 | 有給休暇残数の事前チェック | 高 | 有給休暇を申請する際、申請を提出する前に現在の有給休暇の残日数をシステムが照合し、残日数が不足している場合は申請エラーとすること。 |
| BR-005 | 休暇申請管理 | 休暇申請時の添付ファイル追加 | 中 | 病欠や特別休暇などの場合、必要に応じて診断書や証明書等の添付ファイルを申請に追加できること。なお、添付可能なファイル形式や上限サイズは未定義です。※要確認 |
| BR-006 | 休暇申請管理 | 自身の申請状況・履歴の照会 | 高 | 従業員が、自分が申請した過去の履歴および現在進行中の申請ステータス（申請中、承認、却下等）を一覧で確認・追跡できること。 |
| BR-007 | 休暇申請管理 | 休暇タイプの選択機能 | 中 | 申請時に「有給休暇」「病欠」など適切な休暇タイプを選択できること。選択肢となる具体的な休暇マスタ（慶弔、特別休暇、半日休暇など）の構成は未定義です。※要確認 |
| BR-008 | 休暇申請管理 | 申請済みの休暇のキャンセル・変更ルール | 中 | 申請中または既に承認された休暇申請について、従業員自身によるキャンセルや日程変更を可能とするか、およびその期限に関する業務ルールは未定義です。※要確認 |
| BR-009 | 承認ワークフロー管理 | 直属マネージャーによる承認・却下処理 | 高 | 申請された休暇要求に対し、申請者の直属マネージャーが内容を確認の上、承認または却下を行う。※承認フローが直属マネージャーによる1段階の承認プロセスのみでよいかは要確認。 |
| BR-010 | 承認ワークフロー管理 | 却下理由の必須入力機能 | 高 | マネージャーが休暇申請を却下する場合、却下理由（テキスト）の入力を必須業務ルールとしてシステムで制御する。 |
| BR-011 | 承認ワークフロー管理 | 承認権限の厳格な制御と代理承認の制限 | 高 | 申請者の直属マネージャーのみに承認・却下の権限を付与する。HR管理者は全データの閲覧権限を持つが、マネージャーに代わって代理で承認・却下を行うことは不可とする。※マネージャーが長期不在・退職等で承認できない場合の代替承認ルート・運用ルールについては未定義（※要確認）。 |
| BR-012 | 承認ワークフロー管理 | 承認履歴およびアクションの保持 | 高 | 承認または却下のアクションが実行された際、その処理者、処理日時、および却下時の理由を含むステータス変更履歴を業務証跡として保持する。 |
| BR-013 | 承認ワークフロー管理 | 承認・却下処理におけるステータスの遷移管理 | 中 | マネージャーによる承認・却下確定操作に伴い、対象の休暇申請ステータスを自動的に「承認済」または「却下済」へと遷移させる。※承認待ちのまま一定期間放置された申請に対する督促、自動リマインド、または自動却下の期限ルールの要否については未定義（※要確認）。 |
| BR-014 | 休暇実績・レポート管理 | HR管理者による全社休暇実績・申請情報の照会業務 | 高 | HR管理者が、全従業員の過去および現在の休暇申請状況、承認ステータス、および有給休暇残日数をシステム上で横断的に検索・照会できる業務プロセス。部門別、従業員別、取得期間等の条件でフィルタリング可能とする。 |
| BR-015 | 休暇実績・レポート管理 | 月次休暇実績データの抽出・出力業務 | 高 | HR管理者が、月次の勤怠管理および給与計算の基礎資料として、システムから対象月の休暇取得実績データ（CSVまたはExcel形式 ※要確認）を一括ダウンロードする業務プロセス。出力対象データの確定・締め運用のルール定義が別途必要（未定義 ※要確認）。 |
| BR-016 | 休暇実績・レポート管理 | HR管理者におけるデータ参照権限と代理承認の制限業務ルール | 高 | HR管理者はガバナンス監査および問合せ対応のために全従業員の申請データを参照できるが、承認フローの整合性を保つため、直属マネージャーに代わって申請を承認または却下することはできないものとする業務ルール。 |
| BR-017 | 休暇実績・レポート管理 | 承認済み休暇の変更・取消に関する変更管理業務（※要確認） | 中 | マネージャー承認完了後に、従業員から休暇の急な日程変更や取消要望が発生した場合の対応業務プロセス。HR管理者が管理システム上で直接データの修正・変更を行うのか、または従業員に再度申請を取り直させるのか等の実業務運用フロー（未定義 ※要確認）。 |
